About this tool

This tools is intended to help you get the CTR metric for a given criteria of filters.

I have implemented the logic of both Front and Back-end system to guide you through the usage and better experience during the presentation

The server is waiting for requests in the following format:

  • startTimestamp: This field is mandatory. You have to send it as timestamp of 10 digits and will be returned as date of format of '2016-01-03 13:55:00

  • endTimestamp: This field is mandatory. You have to send it as timestamp of 10 digits

  • aggregation: This field is mandatory. It´s a number between 1 and 60, represented as minutes within an hour and will be the main aggregation factor Since timestamp 0 is 1970-01-01 00:00:00, if you choose the number 6, the values will be divided into partitions of 6 minutes. This form, the transactions will be grouped into bins of 6 to 6 minutes All the values will be group into minimum statTimestamp found of Purchases history that belongs to that partition

  • product: This is an optional parameter. I have not used this field on queries because I didn´t get the logic to query this on table order_items

  • platform: This is an optional parameter. The client may want to filter one of the 3 platforms 'iOS', 'Android', 'MobileWeb'. If empty, values will be grouped onto all device_types


Requesting through HTTP GET

To do a HTTP request to the Server, use the following URL format:

http://localhost:3000/products/summary?startdate=1454284800&enddate=1454371199&aggregation=10&platform=MobileWeb&product=XPTO

Click on the button bellow and you will be redirected to the Front-end form
